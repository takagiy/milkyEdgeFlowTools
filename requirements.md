# milkyEdgeFlowTools -- 要件・仕様書

## 1. 概要

Blender Edit Mode 用のエッジフロー編集ツール集である。v1 では以下の 1 機能を提供する。

**Relax Crossing Flows（交差するフローをリラックス）**: 選択されたエッジループを近似曲線で近似し、その曲線上で頂点をリサンプルする。リサンプル位置は「選択ループに交差するエッジループのフロー（進入方向）が維持される」ように決定され、結果として交差フローがリラックスされる。選択ループ自体の形状は近似曲線によって維持され、頂点の分布だけが変化する。

対応バージョン: Blender 5.0 以降（Extension 形式）

---

## 2. 起動条件と UI

### 2.1 起動条件

アクティブオブジェクトが Mesh であり、Edit Mode に入っていること。編集中の全オブジェクト（マルチオブジェクト編集を含む）が処理対象となる。

### 2.2 メニュー配置

Edit Mode の右クリックコンテキストメニュー（`VIEW3D_MT_edit_mesh_context_menu`）にサブメニュー `milkyEdgeFlowTools` を追加し、その中に本オペレータを配置する。他のメニューには追加しない。

### 2.3 オペレータ

- ID: `mesh.milky_relax_crossing_flows`
- `REGISTER` / `UNDO` 対応。実行後は Redo パネルからパラメータを調整できる。

## 3. 入力の解釈

### 3.1 チェーン分解

選択エッジを連結成分に分解し、成分ごとに独立して処理する（オブジェクトごとにも独立）。

- **開チェーン**: 端点を持つ一続きの選択。ループ一周未満の部分選択も許容される。端点の特別扱いはなく、固定判定（5 章）のみが端点の移動可否を決める。
- **閉チェーン**: 一周つながった選択。周期曲線としてフィットされる。
- **分岐**: 1 頂点に選択エッジが 3 本以上接続する成分は処理不能としてスキップし、警告としてレポートする。他の成分の処理は継続する。

### 3.2 交差エッジの特定

チェーン上の頂点 v に接続するエッジのうち、v におけるチェーンエッジと面を共有するものを**交差エッジ**とする（チェーンエッジ自身は除く）。エッジループの未選択の続き（チェーンエッジと面を共有しない）は交差エッジとみなされない。

交差エッジが片側にしかない頂点はその側のみを使う。交差エッジの構成が想定外（0 本、または面共有関係から側を特定できない）の頂点は外挿対象から除外され、平滑化ソルバー（6 章）が位置を補間する。

---

## 4. 近似曲線

チェーンの頂点列をセントリペタル Catmull-Rom スプラインで補間する。曲線は元の全頂点を通過するため、選択ループの形状は頂点位置において正確に維持される。閉チェーンは周期化される。

曲線は弧長でパラメータ化され、以下の操作を提供する。

- 弧長 s に対応する点の取得
- 元の頂点 i に対応する弧長の取得
- レイ（半直線）との最近接点の弧長の取得

リサンプル後の頂点は曲線上にそのまま配置される。元メッシュ表面への吸着は行わない。

---

## 5. 固定判定（ピン留め）

チェーン上の頂点 v は、v に接続する**交差エッジ**が以下のいずれかを満たす場合、移動しない（ピン留め）。交差エッジがオブジェクトの形状を構成している場合にそれを変形させないためである。選択ループ自体の形状は近似曲線が維持するため、チェーンエッジ側の判定は行わない。

1. **境界エッジ**: 隣接面が 2 未満（開いた縁、ワイヤエッジを含む）
2. **シャープな折れ**: 2 つの隣接面の内角（平坦 = 180°）が `Face Angle Limit`（デフォルト 90°）以下
3. **Sharp マーク**: エッジに Sharp がマークされている

また、`Lock Ends` が有効な場合、開いたチェーンの両端の頂点は上記の判定によらず固定される（閉チェーンには影響しない）。

---

## 6. リサンプル位置の決定

各段階は差し替え可能なモジュールとして実装する（将来、別のリラックス定義や曲線方式に交換できるようにする）。

### 6.1 フロー外挿（データ項）

頂点 v の各サイドについて、交差エッジの先の交差ループを最大 16 エッジまでたどる。

- **外挿パス**: 交差エッジの根本（v の隣の頂点）から、フローの続きを**曲率減衰スパイラル**として延長する。直近 3 点の離散曲率 κ₀ を初期値に、曲率を κ(s) = κ₀·e^(−s/λ) で指数減衰させながら積分する（λ = 平均セグメント長の 3 倍、ただし総回転角 κ₀λ ≤ 90° となるよう頭打ち。これによりパスは決してループバックしない）。減衰しきった後は直線として延長する。点が 2 つ以下、または曲率がほぼ 0 の場合は直線レイに縮退する。
- **目標位置**: 外挿パスと近似曲線との最近接点。その弧長 s\* を目標とする（探索は s_orig ± 1.5×クランプ幅に限定）。
- **サイドの合成**: リング数（たどれたエッジ数）が多い側の s\* を `Side Blend = 0`、少ない側を `1` として線形ブレンドする。デフォルトは 0（多い側 100%）。片側しか無い場合はその側を使う。
- **安全クランプ**: s\* の元位置からの変位は、隣接頂点との元の間隔の 2 倍までにクランプする（レイが曲線とほぼ平行な場合の暴走防止）。

両サイドとも外挿不能な頂点はデータ項を持たない。

### 6.2 平滑化ソルバー

頂点 i の弧長変位 δᵢ = sᵢ − s⁰ᵢ について、次の 2 次形式を最小化する 1 次元問題を解く。

- **データ項**: Σ wᵢ (δᵢ − δ\*ᵢ)²。外挿目標を持つ頂点は wᵢ = 1、持たない頂点は wᵢ = 0。
- **平滑項**: λ Σ (δᵢ₊₁ − δᵢ)²。隣接頂点が同程度に変位するよう拘束し、元の間隔パターンを保存する。λ は `Stiffness`。
- **固定制約**: ピン留め頂点は δᵢ = 0（大重みペナルティで実現）。

固定点の影響は平滑項を通じて周囲へ伝わり、離れるに従い自然に減衰する。これにより「オブジェクト境界のフローの影響は近傍ほど強く、離れると交差フロー維持が優位になる」という挙動が実現される。

開チェーンは三重対角系、閉チェーンは巡回三重対角系となり、それぞれ O(n) で解く。

### 6.3 複数チェーンの適用順

複数のチェーンが選択されている場合、チェーンは順次適用され、後続のチェーンはリラックス済みのジオメトリからフローを外挿する。順序は、ブレンド率が高い側（`Side Blend` ≤ 0.5 ならリング数が多い側、> 0.5 なら少ない側）に見える他の選択チェーンを先に適用するよう、依存関係の安定トポロジカルソートで決定する。循環依存は選択の入力順にフォールバックする。

### 6.4 イテレーション

リラックスパス（外挿 → ソルバー → 再配置）は `Iterations` 回繰り返される。各イテレーションでは全チェーンが 6.3 の順序で再適用され、更新済みのジオメトリからターゲットを再計算する。近似曲線は実行ごとに 1 回だけフィットされ全イテレーションで共有されるため、繰り返してもループの形状はドリフトしない。`Factor` は最後に 1 回だけ適用される。

### 6.5 間隔の保護

交差フローが狭まりながら交差している場合、隣接頂点のターゲットが曲線上のほぼ同じ位置を指し、頂点が過度に密集しうる。これを二段構えで防ぐ。

1. **ソフト層（IRLS）**: ソルバーの解で間隔が `Min Spacing` × 元の間隔を下回ったペアに片側ばね（重み 10）を追加して再求解する（最大 3 回）。詰まりが周囲へ滑らかに再分配される。
2. **ハード層（PAVA 射影）**: 制約 tᵢ₊₁ − tᵢ ≥ β·gᵢ（β = `Min Spacing`、gᵢ = 元の間隔）を満たす最近傍解へ射影する。変数変換により単調回帰に帰着し、PAVA で O(n)・厳密に解く。固定頂点は区間境界として扱い、β = 0 でも追い越し防止の微小下限（平均間隔の 0.1%）は維持される。閉チェーンは最初の固定頂点（無ければ先頭）で切り開いて射影する。

最小間隔を絶対値ではなく元の間隔に対する比率で定めるのは、フローの集束によって元のメッシュ間隔が狭まっている箇所では、それに比例した狭まりが正しい挙動であるため。

### 6.6 強度

最終位置は `Factor` により元の頂点位置と線形ブレンドされる。

## 7. パラメータ一覧（Redo パネル）

| パラメータ | デフォルト | 範囲 | 説明 |
|---|---|---|---|
| Factor | 1.0 | 0.0 -- 1.0 | 元位置と計算結果のブレンド |
| Side Blend | 0.0 | 0.0 -- 1.0 | リング数が多い側 (0) ⇔ 少ない側 (1) のフローのブレンド |
| Face Angle Limit | 90° | 0° -- 180° | 交差エッジを形状エッジと判定する 2 面の内角のしきい値 |
| Stiffness | 1.0 | 0.0 -- 100.0 | 平滑化ソルバーの剛性（固定点の影響範囲） |
| Min Spacing | 0.3 | 0.0 -- 1.0 | 隣接頂点間の間隔の下限（元の間隔に対する割合） |
| Iterations | 1 | 1, 5, 10, 15, 20, 25, 30 | リラックスパスの適用回数 |
| Lock Ends | OFF | -- | 開いたエッジループの両端の頂点を固定 |

## 8. v1 のスコープ外（将来課題）

- UV・頂点ウェイト等の属性補正（Edge Slide の Correct UVs 相当）
- 交差ループ側の頂点の移動（現状は選択チェーン上の頂点のみ移動する）
- リラックス定義の別方式（直交方式、曲げエネルギー最小化など。ソルバー・目標位置計算は差し替え可能な構造とする）
- 選択ループ自体のスムージング（最小二乗 B スプライン等。曲線生成も差し替え可能な構造とする）

## 9. ファイル構成

Extension 形式（Blender 5.0 以降）として、以下で構成される。

- `__init__.py` -- アドオン本体（bpy / bmesh 統合、オペレータ、メニュー、翻訳）
- `core.py` -- 純ロジック（チェーン分解、曲線、外挿、ソルバー。bpy に依存しない）
- `blender_manifest.toml` -- Extension メタデータ（`[build]` で配布 zip の除外パターンを定義）
- `test_core.py` -- core.py のテスト。Blender 外の素の Python で実行可能
- `test_blender.py` -- ヘッドレス Blender で実行する統合テスト
- `requirements.md` -- 本書

配布 zip に含まれるのは `__init__.py` / `core.py` / `blender_manifest.toml` のみ。

## 10. 開発ツーリング

依存管理は uv、タスクランナーは poethepoet（`pyproject.toml` に定義）。開発・CI/CD は Blender 本体に依存しない。

- Python は Blender 5.0 同梱版に合わせて 3.11 に固定（bpy wheel の要求）
- 統合テストは PyPI の公式 `bpy` wheel（5.0.1）に対して実行する
- ビルド・検証は Blender 公式の単体スクリプト `tools/blender_ext.py`（Blender ソース v5.0.1 タグからベンダリング、SHA-256 は `tools/blender_ext.py.sha256`）で行う。出力は `blender --command extension build` と同一
- 実機 Blender での確認用に `test-blender-app` を残している（環境変数 `BLENDER` で場所を指定。デフォルト: `C:/Program Files/Blender Foundation/Blender 5.0/blender.exe`）

| コマンド | 内容 |
|---|---|
| `uv run poe test` | 全テスト（ユニット + bpy 統合） |
| `uv run poe test-core` | ユニットテストのみ |
| `uv run poe test-blender` | bpy wheel に対する統合テスト |
| `uv run poe test-blender-app` | 同じテストを実機 Blender で実行（任意） |
| `uv run poe validate` | Extension マニフェストの検証 |
| `uv run poe build` | 配布 zip を `dist/` にビルド |

Blender のバージョンを上げる際は、`bpy` の依存バージョン・`tools/blender_ext.py`・`blender_version_min` を揃えて更新する。
