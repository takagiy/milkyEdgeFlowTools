# milkyEdgeFlowTools -- 要件・仕様書

## 1. 概要

Blender Edit Mode 用のエッジフロー編集ツール集である。v1 では以下の 1 機能を提供する。

**Relax Crossing Flows（交差するフローをリラックス）**: 選択されたエッジループを近似曲線で近似し、その曲線上で頂点をリサンプルする。リサンプル位置は「選択ループに交差するエッジループのフロー（進入方向）が維持される」ように決定され、結果として交差フローがリラックスされる。選択ループ自体の形状は近似曲線によって維持され、頂点の分布だけが変化する。

対応バージョン: Blender 5.0 以降（Extension 形式）

---

## 2. 起動条件と UI

### 2.1 起動条件

アクティブオブジェクトが Mesh であり、Edit Mode に入っていること。編集中の全オブジェクト（マルチオブジェクト編集を含む）が処理対象となる。

### 2.2 メニュー配置

Edit Mode の右クリックコンテキストメニュー（`VIEW3D_MT_edit_mesh_context_menu`）にサブメニュー `milkyEdgeFlowTools` を追加し、その中に各オペレータ（Relax Crossing Flows、Regenerate Crossing Flows — 11 章、Equalize Loop Spacing — 12 章）を配置する。他のメニューには追加しない。

### 2.3 オペレータ

- ID: `mesh.milky_relax_crossing_flows`
- `REGISTER` / `UNDO` 対応。実行後は Redo パネルからパラメータを調整できる。

## 3. 入力の解釈

### 3.1 チェーン分解

選択エッジを連結成分に分解し、成分ごとに独立して処理する（オブジェクトごとにも独立）。

- **開チェーン**: 端点を持つ一続きの選択。ループ一周未満の部分選択も許容される。端点の特別扱いはなく、固定判定（5 章）のみが端点の移動可否を決める。
- **閉チェーン**: 一周つながった選択。周期曲線としてフィットされる。
- **分岐**: 1 頂点に選択エッジが 3 本以上接続する成分は処理不能としてスキップし、警告としてレポートする。他の成分の処理は継続する。

### 3.2 交差エッジの特定

チェーン上の頂点 v に接続するエッジのうち、v におけるチェーンエッジと面を共有するものを**交差エッジ**とする（チェーンエッジ自身は除く）。エッジループの未選択の続き（チェーンエッジと面を共有しない）は交差エッジとみなされない。

交差エッジが片側にしかない頂点はその側のみを使う。交差エッジの構成が想定外（0 本、または面共有関係から側を特定できない）の頂点は外挿対象から除外され、平滑化ソルバー（6 章）が位置を補間する。

---

## 4. 近似曲線

チェーンの頂点列をセントリペタル Catmull-Rom スプラインで補間する。曲線は元の全頂点を通過するため、選択ループの形状は頂点位置において正確に維持される。閉チェーンは周期化される。

曲線は弧長でパラメータ化され、以下の操作を提供する。

- 弧長 s に対応する点の取得
- 元の頂点 i に対応する弧長の取得
- レイ（半直線）との最近接点の弧長の取得

リサンプル後の頂点は曲線上にそのまま配置される。元メッシュ表面への吸着は行わない。

---

## 5. 固定判定（ピン留め）

チェーン上の頂点 v は、v に接続する**交差エッジ**が以下のいずれかを満たす場合、移動しない（ピン留め）。交差エッジがオブジェクトの形状を構成している場合にそれを変形させないためである。選択ループ自体の形状は近似曲線が維持するため、チェーンエッジ側の判定は行わない。

1. **境界エッジ**: 隣接面が 2 未満（開いた縁、ワイヤエッジを含む）
2. **シャープな折れ**: 2 つの隣接面の内角（平坦 = 180°）が `Face Angle Limit`（デフォルト 90°）以下
3. **Sharp マーク**: エッジに Sharp がマークされている

また、`Lock Ends` が有効な場合、開いたチェーンの両端の頂点は上記の判定によらず固定される（閉チェーンには影響しない）。

---

## 6. リサンプル位置の決定

各段階は差し替え可能なモジュールとして実装する（将来、別のリラックス定義や曲線方式に交換できるようにする）。

### 6.1 フロー外挿（データ項）

頂点 v の各サイドについて、交差エッジの先の交差ループを最大 16 エッジまでたどる。

- **進入方向**: v へ到達する直前の 2 セグメント（交差エッジ自身は含まない）の方向の重み付き平均（近い側 0.7 / 遠い側 0.3）。1 セグメントしか無い場合はその方向。0 セグメント（交差エッジの先に続きが無い）の場合、そのサイドは外挿不能。
- **目標位置**: 交差エッジの根本（v の隣の頂点）を原点とし進入方向へ伸ばしたレイと、近似曲線との最近接点。その弧長 s\* を目標とする。
- **サイドの合成**: リング数（たどれたエッジ数）が多い側の s\* を `Side Blend = 0`、少ない側を `1` として線形ブレンドする。デフォルトは 0（多い側 100%）。片側しか無い場合はその側を使う。
- **安全クランプ**: s\* の元位置からの変位は、隣接頂点との元の間隔の 2 倍までにクランプする（レイが曲線とほぼ平行な場合の暴走防止）。

両サイドとも外挿不能な頂点はデータ項を持たない。

### 6.2 平滑化ソルバー

頂点 i の弧長変位 δᵢ = sᵢ − s⁰ᵢ について、次の 2 次形式を最小化する 1 次元問題を解く。

- **データ項**: Σ wᵢ (δᵢ − δ\*ᵢ)²。外挿目標を持つ頂点は wᵢ = 1、持たない頂点は wᵢ = 0。
- **平滑項**: λ Σ (δᵢ₊₁ − δᵢ)²。隣接頂点が同程度に変位するよう拘束し、元の間隔パターンを保存する。λ は `Stiffness`。
- **固定制約**: ピン留め頂点は δᵢ = 0（大重みペナルティで実現）。

固定点の影響は平滑項を通じて周囲へ伝わり、離れるに従い自然に減衰する。これにより「オブジェクト境界のフローの影響は近傍ほど強く、離れると交差フロー維持が優位になる」という挙動が実現される。

開チェーンは三重対角系、閉チェーンは巡回三重対角系となり、それぞれ O(n) で解く。

### 6.3 複数チェーンの適用順

複数のチェーンが選択されている場合、チェーンは順次適用され、後続のチェーンはリラックス済みのジオメトリからフローを外挿する。順序は、ブレンド率が高い側（`Side Blend` ≤ 0.5 ならリング数が多い側、> 0.5 なら少ない側）に見える他の選択チェーンを先に適用するよう、依存関係の安定トポロジカルソートで決定する。循環依存は選択の入力順にフォールバックする。

### 6.4 イテレーション

リラックスパス（外挿 → ソルバー → 再配置）は `Iterations` 回繰り返される。各イテレーションでは全チェーンが 6.3 の順序で再適用され、更新済みのジオメトリからターゲットを再計算する。近似曲線は実行ごとに 1 回だけフィットされ全イテレーションで共有されるため、繰り返してもループの形状はドリフトしない。`Factor` は最後に 1 回だけ適用される。

### 6.5 順序の保証

解いた後、曲線上の頂点順序が反転（追い越し）しないよう、最小間隔（元の平均間隔の 1% ）を保つ単調化パスを適用する。

### 6.6 強度

最終位置は `Factor` により元の頂点位置と線形ブレンドされる。

## 8. v1 のスコープ外（将来課題）

- UV・頂点ウェイト等の属性補正（Edge Slide の Correct UVs 相当）

## 11. 機能: Regenerate Crossing Flows（交差するフローを再生成）

### 11.1 概要

おおまかに平行な複数のエッジチェーン（**レール**）を選択し、右クリックメニューから起動するモーダルツール。両端のレールに挟まれた面領域（**ストリップ**）を削除し、レールの近似曲線上に交差フローを再生成する。調整モード内でプレビューを見ながらロック・ドラッグで調整し、Enter で確定する。

- オペレータ ID: `mesh.milky_regenerate_crossing_flows`
- 表示名: Regenerate Crossing Flows / 交差するフローを再生成

### 11.2 レールの条件と検証

- レールは互いに交差しない開チェーン群（**v1 は閉ループ対象外**。環状ストリップは将来課題）
- 対象は**アクティブオブジェクトのみ**（本ツールはマルチオブジェクト編集非対応）
- レールの並び順（どれが端か）は空間座標ではなく、交差エッジを辿った隣接関係（トポロジー）で決定する。並びが一本道にならない選択はエラー
- レールの向きは端点間距離の和が最小になるように揃える
- チェーンごとの頂点数は一致していなくてよい。必要なのは「全チェーンが両端で共通のエンド行（交差パス）と交わること」のみ
- **はみ出しの自動トリム**: ループ選択がストリップ角を越えて余分に走った場合など、チェーンがエンド行の外へはみ出していても、エンド行探索は最外レール上の任意の頂点で停止でき（両方向から試行し、全レールとちょうど1回ずつ交わるパスのみ採用）、各チェーンはエンド行まで自動で切り詰められる。トリム後2頂点未満になるチェーンはエラー
- 検証に失敗した場合はエラーをレポートして通常モードのまま。成功したら調整モードに入り初回生成をプレビュー表示する

### 11.3 ストリップの特定と削除

- **特定**: 中間レールに接する面から面領域を flood fill し、両端レールで停止した領域
- **削除**: ストリップ内部の全頂点（付随する辺・面も連鎖削除）。中間レールの頂点も内部扱い。ただし**各レールの端点頂点は保持**する（位置は曲線端と一致するため、ストリップの端の外側にある面は自然に生存する）
- **旧構造は不問**: 内部に三角形・n-gon・ポールが混ざっていてもよい。ただし**両端の列（エンドパス）は交差パスとして辿れる**必要がある（辿れない場合はエラー）。内部に穴（別の境界ループ）がある場合は v1 ではエラー（穴を避けた再生成は将来課題）
- **外側接続**: 両端レールの外側に接続する面は、レールに接していた辺列を新しいレール頂点列で置き換えて **n-gon として貼り直す**

### 11.4 フロー本数

- **ロックなし時**: UI の本数パラメータで決定。初期値は両端レールの元頂点数の平均（四捨五入）、最小値 2（両端フローのみ）
- **チェーンロック時**: ロックチェーンの頂点数がフロー本数を強制し、UI の本数指定は無効化（読み取り専用表示）
- **ロック同士の競合**: 頂点数が一致しないチェーンのロック操作は拒否し、HUD に理由を表示する

### 11.5 分割密度（きれいな分割）

両端レールのサンプル位置は、弧長と曲率の重み付き密度で決める（参考: Curvature based sampling of curves and surfaces, CAGD 2017）。

- 密度 **w(t) = (1 − α) + α·κ̂(t)**。κ̂ は近似曲線の離散曲率を平滑化し平均 1 に正規化したもの（全域でほぼ 0 なら一様密度）。α は `Curvature Bias`（0 = 完全等間隔、デフォルト 0.5）
- 累積 F(t) = ∫w の**等分位を逆引き**して n 点を配置。端点（t = 0, 1）は必ず含む
- **両端レールの対応付け**: 両レールの w(t) を正規化パラメータ上で平均した**共通密度**を使い、同じ分位 t で両レールをサンプリングする（フローの平行性を保つ）

### 11.6 ロックと比率の伝播

- 調整モードで選択チェーンを **Shift+クリック**するとロック/解除。ロックされたチェーンの頂点は調整モード進入前の位置で固定され、再生成で変動しない
- ベース生成は常に**アンカー付き再帰二分ブレンド**の1系統で行う。アンカー（各行の位置が確定しているレール）は次のとおり:
  - **ロックあり**: 全ロックレールがアンカー（行位置 = ロック頂点。本数 = ロック頂点数、曲率密度は不使用）
  - **ロックなし**: **両方の最外レール**をアンカーとし、行位置 = 共通曲率密度（`Curvature Bias` 有効）の分位 × 各レール弧長。本数は UI の Flow Count
- レール列をアンカーレールで**セグメントに分割**し、それぞれ独立に解く:
  - **両端アンカーのセグメント**: 各行の始点終点が確定済み。確定行2本を「始点終点距離を正規化した弦フレーム」（弦からの偏差ベクトルを最小回転＋弦長比スケールでターゲット弦へ輸送）でブレンドし、既知の両端点間に直接フィット。フィットカーブと中間レールの最近接点で行を確定
  - **自由な最外レール**（ロックされていない側）: 最寄りロックレールの弧長比率を写してアンカー化し、両端アンカーの直接フィットに帰着させる（**比率コピー固定**。v0.11〜v0.12 の実モデル比較でレイ照準・曲率密度より良好だったため、v0.14 で切替スイッチ Free Side Fit を廃止して一本化）
  - ブレンド重みはアンカーの弧長比率（両端アンカー時は両側の平均）。行は再帰二分で確定し、位置は確定済みの隣接行間にクランプ
- セグメント継ぎ目（ロックレール上）は C0 接続（角を許容）。ドラッグによる手動制約は後段の伝搬で常に優先
- **生成モード（Generation Mode）**: N パネルの enum で切替。`Blend`（ブレンド・既定）= 上記のアンカー付き二分ブレンド。`Copy Flow Shape`（フロー形状コピー）= 全行を**基準行の平行コピー**として生成する:
  - **基準行** = 制約のある行のうち最後にロック/ドラッグ操作した行。操作履歴がなければ最小インデックスの制約行（通常はデフォルトエンドロックされた行0＝下端行。デフォルトエンドロックはモード進入時の初期化にすぎず、以降は他の制約と同列に扱う）。制約行が1本もない場合のみ切替を拒否し、エラーメッセージを表示して Blend に戻す
  - **アンカーは常に片側のみ**: ロックチェーンが1本あればその knot（本数もそれで固定）、なければ弧長が長い方の最外レールに曲率密度サンプリング（`Curvature Bias` 有効）で Flow Count 本。**複数チェーンのロックは過剰拘束のためエラー**（フロー形状コピーでロックできるチェーンは1本まで）
  - **行の生成**: 各アンカーから**基準行の弦方向（固定・全行共通）**にレイを伸ばし、反対側最外レールとの最近接点を終点にする（中間チェーンがアンカーの場合は両方向へ伸ばして両最外レールに到達）。基準行の弦からの偏差プロファイルを**回転なし・弦長比の等方スケールのみ**で配置し、中間レール頂点はそのカーブへの最近接点として各レール曲線上に取る。全行で形状と向きが一致し、スケールだけが変わる
  - **制約**: コピー生成をベースに、従来どおり頂点制約（基準行のロック、他の行のロック/ドラッグ、端行デフォルトエンドロック）を Influence 付きで伝搬。端の行は境界に馴染み、基準行はロック頂点を厳密に通る。確定は既存処理（端行がレール端に届かない場合は境界頂点の移動で対応）
- 経緯: 比率コピー（v0.7）→ 最外1本ロック限定の二分ブレンド（v0.10、好評）→ 全ケース一般化（本節）。最近接点投影（v0.4.1〜）と端の理想位置δ（v0.8〜v0.9）は試行の結果不採用
- **中間レールには対応を拘束として伝搬させない** — 中間頂点の位置は 11.7 の平滑化のみで決まる（ロック比率は反復の初期値として使うだけ）
- フロー頂点の **Shift+クリック**は「現在位置でロック ⇄ 解除」のトグル（ドラッグなしのアンカー打ちが可能）。ドラッグも同様に通過点制約を作る。制約は 11.7 の伝搬により近隣フローへ減衰しながら波及する
- **端フローのデフォルトロック**: 調整モード進入時、最初と最後のフロー行の全頂点にレール端点位置（モード進入前の位置）で頂点制約を自動付与する（本数変更時も再付与）。ロックのままなら確定時はレール端点頂点を再利用（従来互換）。Shift+クリックで個別解除でき、解除された端頂点は通常のフロー頂点として動く。**レール端点頂点のトポロジー保持（外側接続の維持）は不変**で、端フロー行が動いた場合は、レール端点頂点とエンドパスの中間頂点を新しい行位置へ**移動**する（中間頂点は隣接レール間の新しい行セグメント上へ元の間隔比率で再配置）。外側の面は頂点を共有したまま追従し、n-gon の帯は張らない（帯方式は v0.12 まで。移動方式に変更）

### 11.7 フロー生成（レール拘束平滑化）

1. フロー i の頂点を全レールの近似曲線上に置く。初期位置は各レールの弧長比率 tᵢ の点
2. 中間レール上の頂点を、**自分のレール曲線に沿ってスライドさせて**フローポリラインの曲げ（|Qⱼ₋₁ − 2Qⱼ + Qⱼ₊₁|² の和）を最小化する。実装は「隣 2 頂点の中点にレール曲線上で最も近い点へ寄せる」反復（レール拘束付きラプラシアン平滑化）
3. 両端点とロック/ドラッグ済み頂点は固定。フローごとに独立で、頂点は常にレール曲線上に乗る（3D 交点計算は不要）

**制約の伝搬**: 生成は「ベース（頂点制約なし。チェーンロックと比率コピーを含む）→ 制約付きフローを個別に再平滑化 → 変位の伝搬 → 合成」の順で行う。各レール上のフロー頂点列について、制約付きフロー行の変位 δ を固定点とし、自由な行は「δ = 0 への弱いアンカー（重み 1）＋隣接フロー間の平滑項 λ = Influence²」の 1 次元最小二乗で解く。これにより制約の影響が約 `Influence` 本のフローにわたって減衰しながら波及する（Influence = 0 で制約付きフローのみ）。端フローのデフォルトロックはベース（比率の端 = 0/1 = レール端点）と一致するため通常は変位を生まないが、伝搬の固定点として参加する（ユーザー制約の影響は端の行を越えて動かせない）。端の理想位置から δ を作る方式は 2 案（最近接点投影・内部リズム外挿）を試行したがいずれも見た目が合わず、不採用（v0.9.1 で revert）。合成後は各レールで最小間隔（0.1%）の単調化を適用する

## 12. 機能: Equalize Loop Spacing（ループ間隔を均一化）

### 12.1 概要

平行な2本のエッジループを選択し、ループ間の**垂線距離**（頂点から相手ループの近似曲線へ下ろした垂線の長さ）を全周で均一な値 W に揃えるモーダルツール。頂点は**自分のブリッジエッジ（rung）の方向に沿ってのみスライド**するため、ループをブリッジする辺の向き（フロー）は保存される。rung が斜めの箇所では rung 自体は W より長くなる。

- オペレータ ID: `mesh.milky_equalize_loop_spacing`
- 表示名: Equalize Loop Spacing / ループ間隔を均一化
- メニュー: milkyEdgeFlowTools サブメニュー

### 12.2 入力条件

- 選択がちょうど2本の分岐なしエッジチェーンに分解できること（それ以外はエラー）
- 2本の開閉が一致していること（開＋閉の混在はエラー）。**開・閉両対応**
- 2ループが1列の面帯で直接ブリッジされ、各頂点にちょうど1本の rung エッジがあり、対応が全単射であること（満たさなければエラー）
- 対象はアクティブオブジェクトのみ

### 12.3 動く側（ピボットポイント準拠）

`tool_settings.transform_pivot_point` を参照する。

- **アクティブ要素**: アクティブ要素（頂点/エッジ）を含むループを**固定**し、反対側のループが動く（変形時にアクティブが動かないのと同じ意味論）。アクティブがどちらのループにも無い場合は中点扱いにフォールバック
- **それ以外（中点含む）**: 各 rung の中点は不動のまま、**両ループが対称に**開閉する（中間カーブから両側へ W/2 ずつ）

### 12.4 位置の計算

- 基準曲線 = 固定ループ（対称時は rung 中点列）の CatmullRom 近似曲線
- 各頂点の新位置 = 基準点 + rung 方向 × スライド量 s。s は「新位置から基準曲線への垂線距離 = 目標値」となるよう、接線フレームの初期解を実距離測定で数回反復して精密化する
- **安全クランプ**: rung がループ接線とほぼ平行で垂線成分が小さい箇所は、スライド量を目標距離の2倍までにクランプ。rung 長ゼロ（重複頂点）の対は移動しない
- トポロジー・UV 等の属性は変更しない（頂点移動のみ）

### 12.5 操作（Shrink/Fatten 風軽量モーダル）

| 操作 | 動作 |
|---|---|
| 起動 | 現在の平均垂線距離を W としてただちに均一化プレビュー |
| マウス移動 | W を増減（ビューのスケールに追従。Shift で 0.1 倍の精密調整） |
| 左クリック / Enter | 確定（Undo 1 ステップ、Redo パネルに Distance） |
| 右クリック / Esc | キャンセル（元の頂点位置に復元） |
| ビュー操作 | PASS_THROUGH |

- HUD（POST_PIXEL）: 現在の W とモード（固定側 / 対称）、キーガイド
- オーバーレイ（POST_VIEW）: 固定ループを黄色で表示（対称時はなし）
- Redo パネル: **Distance**（0 = 現在の平均距離で自動。確定時の値が入る）。Factor は設けない
